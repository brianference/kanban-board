# FlowBoard v4.0.0 — Major release

**Release date:** 2026-07-19  
**Codename:** Cloud kanban polished  
**Production:** https://kanban-board-public.pages.dev

## Summary

FlowBoard v4 is the production multi-user kanban on **Cloudflare Pages + D1**, with a unified product chrome, legacy-inspired board cards, global search, dark mode, and a full attachment pipeline that actually stores and serves images.

| Layer | Stack |
|-------|--------|
| Frontend | Vite · React 18 · TypeScript · Tailwind (layout) · design tokens · React Router |
| Backend | Cloudflare Pages Functions |
| Database | Cloudflare D1 (SQLite) |
| Auth | JWT httpOnly cookie `fb_token` + PBKDF2 (Web Crypto) |
| Deploy | Type B direct upload (`wrangler pages deploy client/dist --branch main`) |

## Highlights

### Platform
- Multi-user accounts: register, login, logout, session cookie
- Projects with templates; boards with five columns by default
- Tasks: create, edit, move (drag), soft-delete
- Checklist, comments, tags, priority, due dates
- Image attachments (PNG/JPEG/GIF/WebP) stored in D1 as base64, served with auth
- Global header search with live results + full search page
- Contact form persistence (`contact_messages`)
- Tenancy isolation (membership-enforced)
- Dark / light theme (`kb-theme`, moon toggle)

### Design
- Unified header · page · footer surface language (no mismatched chrome)
- Board workspace panel: toolbar + filters + columns in one card
- Card look inspired by legacy Task Board: solid priority pills, ID badge, column dots, “days in stage”
- Current features kept: drag grip, checklist/comment/attachment counts, thumbs
- Inter product typography; responsive columns (tabs on mobile, no forced side-scroll)

### Quality
- `scripts/e2e-prod.mjs` — 66+ production checks including PNG magic-byte round-trip
- `npm run test:e2e` defaults to production pages.dev
- Migrations `0001`–`0004` (incl. contact + attachment tables)

## Breaking changes (from v3.0 Express path)

| v3.0 | v4.0 |
|------|------|
| Express + better-sqlite3 | Cloudflare Pages Functions + D1 |
| bcrypt | PBKDF2 (edge-safe) |
| Local `server/data/flowboard.db` | Managed D1 `kanban-board-db` |
| File uploads on disk | D1 blob/base64 storage |
| Railway / Node host | Pages Type B deploy only |

The Express `server/` tree remains in the repo as an optional local path; **production is Cloudflare Pages**.

## Deploy

```bash
# Apply D1 migrations when schema changes
npm run db:migrate

# Secret (once)
npx wrangler pages secret put JWT_SECRET --project-name kanban-board-public

# Build + production deploy
npm run deploy
# → wrangler pages deploy client/dist --project-name kanban-board-public --branch main
```

Requires `CLOUDFLARE_API_TOKEN` + `CLOUDFLARE_ACCOUNT_ID`.

## Ops notes

1. **JWT_SECRET** — production Pages secret (32+ random chars).
2. **D1** — binding `DB` in `wrangler.toml` (`kanban-board-db`).
3. **Images** — max ~900KB per image (D1 value limits); re-upload any pre-v4 broken thumbs.
4. **git push does not deploy** — always `wrangler pages deploy` after release.
