# FlowBoard v3.0.0 — Major release

**Release date:** 2026-07-19  
**Codename:** Full-stack foundation

## Summary

FlowBoard is now a true full-stack application:

| Layer | Stack |
|-------|--------|
| Frontend | Vite + React 18 + TypeScript + Tailwind CSS + React Router |
| Backend | Node.js + Express |
| Database | SQLite via better-sqlite3 |
| Auth | Register/Login with JWT (httpOnly cookie) + bcrypt password hashing |

The previous Cloudflare Workers/D1 SPA remains in git history under older commits and `legacy/` / `functions/` folders for reference, but **v3 production runtime is Express + SQLite**.

## Highlights

- Premium sticky header with FlowBoard logo (inline SVG, fast on mobile)
- Clean multi-column footer (Product / Company / Legal)
- Pages: Home, About, Terms, Privacy, Contact, Login, Register, Dashboard, Board, Task detail, Search
- Plain-language Privacy Policy and Terms & Conditions
- Global search (keyword, priority, due, project)
- Task detail page with checklist, comments, image upload
- Confirm dialogs before deleting projects/tasks
- Loading states and clear API error messages
- SEO: meta tags, Open Graph, Twitter cards, JSON-LD, robots.txt, sitemap.xml
- Helmet security headers, rate limiting, Zod validation
- No “made with Grok” branding

## Breaking changes

- API is Express at `/api/*` (not Cloudflare Pages Functions)
- Sessions use cookie name `fb_token` (JWT)
- Database is a local/server SQLite file (`server/data/flowboard.db`)

## Upgrade / run

```bash
cd server && npm install
cd ../client && npm install && npm run build
cd ../server && JWT_SECRET=your-secret NODE_ENV=production npm start
```

## Critical ops notes

1. Set a strong `JWT_SECRET` (32+ random chars) in production.
2. Persist `server/data/` and `server/uploads/` volumes on the host.
3. Set `CLIENT_ORIGIN` if frontend is on a different origin (CORS + cookies).
4. For cross-site cookies in production, ensure HTTPS (`secure` cookies).
